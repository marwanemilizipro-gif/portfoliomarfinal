Déploiement Automatisé d'Images OS
Serveur FOG – Lycée Jacques Brel
Contexte :

Le lycée avait besoin d'une solution pour réinstaller rapidement les salles C207 et C208 avec une image système propre (Windows ou Linux) via le réseau. Le projet s'est déroulé en deux phases : maquettage dans un réseau isolé sur Proxmox (192.168.2.x) derrière IPFire, puis migration en production sur le réseau administrateur du lycée.

Architecture & Installation :

Serveur FOG déployé sur une VM Debian 13 (Trixie) hébergée sur le cluster Proxmox du lycée, avec adresse IP fixe 172.28.30.15. Le DHCP est géré par le contrôleur de domaine Windows (Active Directory) — pas de DHCP FOG activé pour éviter les conflits réseau.

Démarrage PXE :

Configuration des options DHCP sur le serveur Windows AD : option 66 (Boot Server) pointant vers 172.28.30.15, option 67 (Boot File) avec ipxe.efi pour les postes UEFI récents et undionly.kpxe pour la compatibilité Legacy/BIOS. Désactivation du Secure Boot et de BitLocker sur les postes clients.

Masterisation des images :

Image principale Windows 11 créée avec procédure Sysprep (sysprep /generalize /oobe) pour supprimer les SID et éviter les conflits lors du déploiement multi-postes. Capture en mode "Single Disk – Resizable" pour compatibilité avec différentes tailles de disques. Images Linux Mint et Ubuntu configurées avec les logiciels open-source des TP. Extension du stockage à 282 Go via GParted à chaud.

FOG Project PXE Boot Debian 13 Sysprep Windows 11 Linux Mint Proxmox Active Directory GParted

Veille Technologique

La Conteneurisation

Docker & Kubernetes

Pourquoi ce choix ?

L'industrie informatique délaisse progressivement les machines virtuelles traditionnelles, lourdes et gourmandes en ressources, au profit des conteneurs légers et rapides. Un conteneur Docker démarre en quelques secondes là où une VM nécessite plusieurs minutes. Cette révolution redéfinit le déploiement et la gestion des applications modernes.

Mon Approche

Je m'intéresse au fonctionnement de Docker pour le déploiement d'applications conteneurisées et à l'orchestration avancée via Kubernetes. Je surveille les enjeux de sécurité liés à ces environnements (isolation des conteneurs, gestion des images, vulnérabilités CVE) et les tendances cloud-native.

DOCKER
Conteneurisation

Empaqueter une application avec toutes ses dépendances dans un conteneur isolé, portable et léger.

KUBERNETES
Orchestration

Automatiser le déploiement, la mise à l'échelle et la gestion de conteneurs en production sur plusieurs serveurs.

SÉCURITÉ
Isolation & CVE

Surveillance des vulnérabilités sur les images Docker publiques et bonnes pratiques de durcissement des conteneurs.

LIEN AVEC LE MÉTIER SISR

La conteneurisation s'inscrit directement dans les missions SISR : gestion de l'infrastructure, haute disponibilité des services, et sécurisation des environnements. Kubernetes remplace progressivement les VM dans les datacenters modernes — une compétence clé pour un technicien infrastructure de demain.
